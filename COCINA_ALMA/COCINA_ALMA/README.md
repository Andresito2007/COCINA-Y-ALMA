Contexto o situación problemática: En una empresa de comercio electrónico que vende productos tecnológicos, se ha identificado un problema relacionado con la falta de integración y automatización en la gestión de datos críticos. Actualmente, el sistema utiliza procesos manuales para actualizar el inventario después de cada compra, lo que genera errores frecuentes, como desabastecimiento de productos o inventario negativo. Además, la generación de reportes clave, como la identificación de los productos más vendidos o el análisis de ingresos mensuales, es lenta y propensa a errores debido a la falta de consultas optimizadas.
Por ende, para resolver estas deficiencias se requiere diseñar y desarrollar una base de datos eficiente y escalable utilizando Oracle 21C.
Pasos:
1.0 CREACION DEL USUARIO Y SUS PERMISOS
1. Diseño de la base de datos: 
Crear un script SQL utilizando DDL para definir las siguientes tablas: 
Clientes: con información básica de los clientes (ID, nombre, correo, teléfono).
Pedidos: para registrar pedidos realizados por los clientes (ID, fecha, total, cliente_id).
Productos: con detalles de los productos ofrecidos (ID, nombre, precio, categoría_id, proveedor_id).
Categorías: para clasificar los productos (ID, nombre).
Proveedores: con información sobre los proveedores (ID, nombre, contacto).
2. Población de datos: 

Insertar datos representativos en cada una de las tablas creadas utilizando comandos DML.
Asegurarse de tener datos suficientes para realizar consultas y pruebas efectivas.
3. Consultas SQL:  Elaborar consultas, como por ejemplo: 

Pedidos por cliente: Generar un reporte del total de pedidos y monto acumulado por cliente.
Productos más vendidos: Listar los productos más vendidos con sus respectivas cantidades y ganancias totales.
Inventario actual: Mostrar productos con bajo stock (< 5 unidades).
Reporte de ingresos: Calcular ingresos totales por mes y año.
Producto: Un archivo de texto con extensión .txt o .sql que debe incluir:
Definición de tablas y relaciones (DDL).
Inserción y manipulación de datos simulados (DML).
Consultas SQL optimizadas para extracción y análisis de datos.
Parámetros de formato de presentación:
Archivo en formato .txt o .sql:
El archivo debe estar bien estructurado, con secciones claramente delimitadas para: 
Definición de tablas y relaciones (DDL)
Inserción y manipulación de datos (DML)
Consultas SQL
Documento PDF con el registro detallado de resultados y pruebas realizadas.
